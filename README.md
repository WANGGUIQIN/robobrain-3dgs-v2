# RoboBrain-3DGS V2

Vision-Language planner for robotic manipulation with **causal reasoning + lifted goal predicates + textual affordance regions**.

The model emits a structured plan from a scene image and a task instruction:

```
<think>SCENE ANALYSIS — ... STRATEGY — ... AFFORDANCE REASONING — ...</think>
Goal: on(red_mug, blue_plate); stable(red_mug)
Scene: red_mug, blue_plate, table
Step 1: reach(red_mug)
  affordance_region: the body of the red mug
  contact: gripper_state(open) [progress]
  spatial: distance(gripper, red_mug, <, 0.03) [completion]
  direction: approach_axis(0, 0, -1) [progress]
  done_when: distance(gripper, red_mug) < 0.03
Step 2: grasp(red_mug)
  ...
<END_OF_PLAN>
```

A downstream open-vocabulary grounding model (Lang-SAM) resolves each `affordance_region` description to image coordinates at execution time; the controller (MPC) consumes the constraints and `done_when` predicates.

## What's new vs V1

| | V1 | V2 |
|---|---|---|
| Reasoning | post-hoc rationalization | causal `<think>` block before plan |
| Goal | implicit (last `done_when`) | explicit lifted-predicate `Goal:` line |
| Affordance grounding | LM emits `[u, v]` text tokens | LM emits `affordance_region` text, Lang-SAM grounds at runtime |
| Step constraints | flat | categorized: contact / spatial / pose / direction / safety, with `[role]` labels |
| Step ordering | hard-coded sequence | `done_when` evaluated continuously; order-invariant when goal allows |

## Layout

```
config/        Training configs (LoRA / full FT × A100 / B200 × planning)
data/          Data loaders + dataset splits (processed data is NOT committed)
models/        Model code (VLM backbone, 3D Gaussian branch, fusion)
utils/         Prompt templates, parser, chat-template tokenization
scripts/       Data preparation + V2 plan generation (generate_plans_v2.py)
docs/          Design notes
tests/         Unit tests
train.py       Training entrypoint (DeepSpeed)
inference_3dgs.py  Inference entrypoint
```

## Setup

```bash
pip install -r requirements.txt
```

Required external resources (not in this repo):

- **Base model weights** — RoboBrain2.5-8B-NV (download separately; configure via `model.base_model:` in the training yaml)
- **Processed dataset** — episodes with `rgb_0.png`, `depth_0.npy`, `meta.json`, `plan.json`. The dataset can be generated by running `scripts/generate_plans_v2.py` against an OpenAI-protocol endpoint.

## Generate V2 plans

```bash
export OPENAI_API_KEY=sk-...   # NEVER commit; only as env var
export OPENAI_BASE_URL=https://your-endpoint/v1
python scripts/generate_plans_v2.py --all --workers 4
```

Writes `plan.json` per episode under your processed-data root.

## Train

```bash
# LoRA on 4× B200 180GB
deepspeed --num_gpus=4 train.py \
  --config config/train_lora_b200_planning_3epoch.yaml

# Full FT on 4× B200 180GB (baseline)
deepspeed --num_gpus=4 train.py \
  --config config/train_full_b200_planning.yaml

# LoRA on 2× A100 80GB
deepspeed --num_gpus=2 train.py \
  --config config/train_lora_a100_planning_3epoch.yaml
```

Each yaml has a memory-budget breakdown in its header comments.

## Inference

```bash
python inference_3dgs.py \
  --checkpoint /path/to/lora_checkpoint \
  --image scene.png \
  --task "Stack the blue cups."
```

The output is parsed into a structured plan dict via `utils.prompt_utils.parse_planning_output` (V2 + V1 back-compat).

## Upgrading an existing V1 dataset to V2 (no RGB re-download)

If you already have the V1 processed dataset on a target machine, grab the V2
plans bundle (~23 MB) from the project's Google Drive and run the alignment
script. Only the per-episode `plan.json` files are touched; the V1 `plan.json`
is preserved as `plan_v1.json`. RGB / depth / meta are untouched.

```bash
# 1. Download the V2 plans bundle from gdrive:RoboBrain3DGS/datasets/
#    -> robobrain-3dgs-v2-plans.tar.zst  (23 MB)

# 2. Run the alignment in place:
python scripts/align_v2_plans.py \
  --v1-root /path/to/v1/data/processed \
  --plans-source robobrain-3dgs-v2-plans.tar.zst

# 3. (optional) Preview first:
python scripts/align_v2_plans.py \
  --v1-root /path/to/v1/data/processed \
  --plans-source robobrain-3dgs-v2-plans.tar.zst \
  --dry-run
```

Idempotent: a second run sees the already-V2 plans and skips them.
